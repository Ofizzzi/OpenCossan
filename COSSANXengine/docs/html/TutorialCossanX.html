CossanX object

The object CossanX is a handle object stored in a global variable named COSSANX. It is usually initalizated automatically at the matlab start up. In order to verify that Cossan-X has been initializad correcly declare the global variable COSSANX and the display it:

global COSSANX
assert(~isempty(COSSANX),'COSSANX:TutorialCossanX','COSSAN-X toolbox not initialized')

display(COSSANX)

% Now the field name and the property of the object are accessible. For instance
% the field name are:
Sfieldnames=fieldnames(COSSANX)
% The constrains object requires the fieds Linequality to define the type
% of constaints.

% We can now see the content of each field, for instance
SmatlabPath=COSSANX.SmatlabPath

Using static Method

The CossanX object provides a number of different static methods can be used directly without accessing to the global variable.

% the installation path of the Cossan toolbox can be retrieve using the
% following method CossanX.getCossanRoot while
display(CossanX.getCossanRoot)

/home/ep/workspace/COSSAN-X_SVN/COSSANXengine

Change verbosity level

store old verbosity level

NoldLevel=CossanX.getVerbosityLevel;
CossanX.setVerbosityLevel(1);
% The verbosity level is setted to 1, minimal output are shown in the
% console. For instance we can see the output of the object CossanX
display(COSSANX)
% Reset verbisity level to 3 (very verbose)
% Change verbosity level
CossanX.setVerbosityLevel(3)
display(COSSANX)

% Restore original Analysis Name
CossanX.setVerbosityLevel(NoldLevel);

Change Analysis Name

store old analysis name

Sanalysis=CossanX.getAnalysisName;
CossanX.setAnalysisName('New Analysis Name');
% Show analysis name
disp(CossanX.getAnalysisName)
% Restore original Analysis Name
CossanX.setAnalysisName(Sanalysis);

New Analysis Name

Change Working Path

the working path (i.e. the path where all the outputs are stored) using the method CossanX.getCossanWorkingPath

% store old working path
SoldPath=CossanX.getCossanWorkingPath;
CossanX.setWorkingPath([filesep 'tmp']);
% Show working path
disp(CossanX.getCossanWorkingPath)
% Restore original Path
CossanX.setWorkingPath(SoldPath);

/tmp

Random Number Generator

CossanX used as default random number generator the Mersenne Twister algorithm (mt19937ar) it is possible to reinitialize the random number generator using the static method CossanX.resetRandomNumberGenerator

CossanX.resetRandomNumberGenerator(13314)
% Now we sample using the matlab function rand
Vsamples1=rand(4,1)
% if we reset the random number generator and then we sample again we obtain the
% same sample values
CossanX.resetRandomNumberGenerator(13314)

Vsamples2=rand(4,1)

assert(all(Vsamples1==Vsamples2),'Tutotiral:CossanX','RandomNumber Generator not reinitialized')

% To change the random number generator it is necessary first to create a new
% stream (see matlab documentation for randstream)

Vsamples1 =

     7.966076049624990e-01
     8.125816715808279e-01
     9.656726558193522e-01
     3.595084982941822e-01


Vsamples2 =

     7.966076049624990e-01
     8.125816715808279e-01
     9.656726558193522e-01
     3.595084982941822e-01

Set the RandomNumberGenerator

newStream = RandStream('swb2712','Seed',2515);
RandStream.setDefaultStream(newStream);

Create matlab start up file

CossanX provides a static method to create the matlab startup file the automatically initialize the Cossan Engine Toolbox at matlab start up. The static method CossanX.createStartupFile will generate the file for you. A backup copy of old startup file is create automatically.

Remove CossanX toolbox

The CossanX toolbox can be removed from the matlab path using the static method CossanX.removePath
